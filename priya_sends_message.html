💡 Priya's phone doesn't "send text." It converts every character into a number, turns those numbers into binary (0s and 1s), wraps them in packets, and fires them through routers all the way to Raj's phone. Let's follow each step.
Step 02
Breaking into individual characters
The phone looks at the message one character at a time — including spaces and punctuation. Every single character gets its own turn.
📌 The computer sees characters, not words. "Hi Raj!" is not one thing — it is 7 separate characters, each processed independently.

Step 03
ASCII lookup — each character → a number
The phone opens the ASCII table and looks up every character. "H" → 72, "i" → 105, " " → 32, "R" → 82 and so on. Now the message is just a list of numbers.
📖 ASCII is a universally agreed codebook. Priya's Mumbai phone and Raj's Delhi phone both have the exact same table. That's how they understand each other.

Step 04
Converting numbers to 0s and 1s
Each ASCII number is written in binary. "H" = 72 = 01001000. These bits are what literally travel through the wire / fibre / Wi-Fi as electrical pulses or light flashes.
⚡ In a copper wire: 1 = high voltage, 0 = low voltage. In fibre optic cable: 1 = light pulse ON, 0 = light OFF. In Wi-Fi: two different radio wave patterns represent 0 and 1.

Step 05
Wrapping bits into packets
The binary stream is chopped into small packets. Each packet gets a header — like addressing an envelope — telling routers where to send it and how to reassemble everything at Raj's end.
📦 Why packets? If the full message was sent as one piece and something went wrong mid-way, everything would need to be re-sent. With packets, only the lost packet is re-sent. Also different packets can travel through different routes simultaneously — much faster!

Step 06
Packets travel through routers
Each packet is passed from router to router. Every router reads the destination IP (Raj's IP: 203.0.113.45) and decides the best next hop. Different packets may take different paths — and that's perfectly fine!
🔀 This is packet switching. Each router is like a post office that reads the address and decides which truck to put the packet on. The internet automatically reroutes if one path is congested or broken.

Step 07
Raj's phone reassembles the message
All packets arrive at Raj's phone (possibly out of order). TCP uses the sequence numbers in each packet's header to sort them, reassemble the binary, look up ASCII, and display the original text.
🔧 If any packet is missing or corrupted (CRC check fails), Raj's phone sends a NAK back to Priya's phone asking for that specific packet again. The entire message does NOT need to be re-sent — only the missing piece.
