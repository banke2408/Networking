Step 02 — ASCII Lookup
Each character has a number
ASCII (American Standard Code for Information Interchange) maps every printable character to a decimal number between 0–127. 'A' = 65, 'a' = 97, space = 32. Your computer looks up this table for every single character.
📖 Think of ASCII like a secret codebook both the sender and receiver agree on. The sender converts text → numbers, sends numbers, receiver converts numbers → text back.

Step 03 — Binary Encoding
Converting to 0s and 1s
The ASCII number is then written in binary — base-2 number system using only 0 and 1. Each character becomes exactly 8 bits (1 byte). These bits are what actually travel through wires, fibre, and radio waves.
⚡ In a wire: 1 = high voltage (on), 0 = low voltage (off). In fibre optic: 1 = light pulse, 0 = no light. In Wi-Fi: different radio frequencies represent 0 and 1.

Step 04 — Packet Formation
Breaking data into packets
The binary stream is split into small chunks called packets. Each packet gets a header — like an envelope label — with the sender's IP, receiver's IP, packet number, and other control info.
📦 Packets are typically up to 1500 bytes (MTU — Maximum Transmission Unit). Splitting allows different packets to take different routes, and if one is lost, only that packet is re-sent — not the whole message.

Step 05 — Network Routing
Packets find their own path
Each packet is handed from router to router. Every router reads the destination IP in the header and decides the best next hop — like asking for directions at each town. Different packets can take completely different routes!
🔀 This is called packet switching — invented in the 1960s. It's why the internet is so resilient: if one path fails, packets automatically find another. Your video call packets might travel through 15 different countries!

Step 06 — Reassembly
Putting it back together
At the destination, TCP collects all packets. Because each packet carries its sequence number, TCP can sort them into the correct order — even if they arrived shuffled — and reconstruct the original message perfectly.
🔧 If any packet is missing or corrupted (checksum fails), TCP sends a NAK (negative acknowledgement) asking the sender to re-transmit just that one packet. That's why TCP is called reliable transport.
