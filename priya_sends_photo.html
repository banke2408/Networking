Step 02
The image is a grid of pixels
Every digital image is just a rectangular grid of tiny coloured dots. Each dot is called a pixel (picture element). Hover or click a pixel to inspect it.
🔍 A 1920×1080 HD image has 2,073,600 pixels. A 4K image has over 8 million. Your phone's camera captures them all in a fraction of a second!

Step 03
Each pixel = 3 numbers (R, G, B)
Every pixel is stored as three numbers: Red, Green, Blue — each between 0 and 255. By mixing different amounts, any colour in the world can be represented.
🎨 0 = none of that colour · 255 = full intensity. White = (255,255,255). Black = (0,0,0). Pure red = (255,0,0). This is called the RGB colour model — the same one used by every screen.

Step 04
Each RGB value → 8 binary bits
Each of the three numbers (R, G, B) is converted to 8-bit binary. So every pixel = 24 bits = 3 bytes. These are the 0s and 1s that travel through wires and fibre optic cables as electrical pulses and light flashes.
⚡ In fibre optic cable: 1 = laser pulse ON, 0 = laser pulse OFF. These pulses travel at ~200,000 km/s — two-thirds the speed of light! Mumbai to Delhi in microseconds.

Step 05
Compression — shrinking before sending
A raw photo is too large to send quickly. Compression algorithms (like JPEG) find patterns and remove redundant data — making the file much smaller while keeping it recognisable.
🗜️ JPEG works by grouping pixels into 8×8 blocks and discarding detail your eye can't easily notice (high-frequency changes). A 5MB photo becomes ~300KB — 94% smaller! That's why your WhatsApp photos load fast.

Step 06
Image data split into packets
The compressed binary data is cut into small packets. Each packet carries a chunk of pixel data plus a header with Priya's IP, Raj's IP, and a sequence number so the image can be reconstructed correctly on arrival.
📦 A 300KB photo = ~300,000 bytes. Each packet holds ~1460 bytes of data. So the photo needs about 205 packets to travel from Priya to Raj — and they can all take different routes simultaneously!

Step 07
Packets route across the internet
Each image packet hops from router to router — reading the destination IP (Raj's phone) at every step. Some packets go via one path, others via another. All converge at Raj's phone.
🌐 Your image might pass through routers in Mumbai → Pune → Hyderabad → Chennai → Delhi or go via an undersea cable through Singapore. The internet finds the fastest available path automatically.

Step 08
Raj's phone reassembles the image
All packets arrive at Raj's phone. TCP sorts them by sequence number, joins the binary data together, runs JPEG decompression to reconstruct pixel RGB values, and draws the image on screen pixel by pixel.
🖼️ The receiver reconstructs the image tile by tile. If even one packet is missing, TCP asks Priya's phone to re-send just that packet. The image won't display until all pieces are in place.
